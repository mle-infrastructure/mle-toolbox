    MLE-Toolbox Overview

    What Does The mle-toolbox Provide?

    1. API for launching jobs on cluster/cloud computing platforms (Slurm, GridEngine, GCP).
    2. Common machine learning research experiment setups:
      • Launching and collecting multiple random seeds in parallel/batches.
      • Hyperparameter searches: Random, Grid, SMBO, Population-Based Training.
      • Pre- and post-processing pipelines for data prep/result visualization.
    3. Automated report generation for hyperparameter searches.
    4. Storage of results and database in Google Cloud Storage Bucket.
    5. Resource monitoring with dashboard visualization.

    5 Steps To Get Started

    1. Follow the installation instructions and set up your credentials/configurations.
    2. Read the docs to learn about the toolbox and .json & .yaml configuration files.
    3. Watch the YouTube Tutorials series for a hands-on walkthrough.
    4. Check out and re-run the examples to get comfortable.
    5. Run your own experiments using the template files, project and mle run.

    Core Commands of the Toolbox

    You are now ready to dive deeper into the specifics of job configuration and can start running your first experiments from the cluster (or locally on your machine) with the commands:

    Command Description
    ⏳ mle init Start up an experiment.
    🚀 mle run Setup of credentials & toolbox settings.
    🖥️ mle monitor Monitor resource utilisation.
    📥 mle retrieve Retrieve an experiment result.
    💌 mle report Create an experiment report with figures.
    🔄 mle sync-gcs Extract all GCS-stored results to your local drive.
